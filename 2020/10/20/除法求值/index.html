除法求值

题目

本题来源于leetcode,运用基本的图论算法都可以解决，但是需要一定的熟练度才能保证在编写代码的过程中少出bug.

题目如下：

给出方程式 A / B = k, 其中 A 和 B 均为用字符串表示的变量， k 是一个浮点型数字。根据已知方程式求解问题，并返回计算结果。如果结果不存在，则返回 -1.0。

输入总是有效的。你可以假设除法运算中不会出现除数为 0 的情况，且不存在任何矛盾的结果。

示例 1：

输入：equations = [[“a”,”b”],[“b”,”c”]], values = [2.0,3.0], queries = [[“a”,”c”],[“b”,”a”],[“a”,”e”],[“a”,”a”],[“x”,”x”]]
输出：[6.00000,0.50000,-1.00000,1.00000,-1.00000]
解释：
给定：a / b = 2.0, b / c = 3.0
问题：a / c = ?, b / a = ?, a / e = ?, a / a = ?, x / x = ?
返回：[6.0, 0.5, -1.0, 1.0, -1.0 ]

示例 2：

输入：equations = [[“a”,”b”],[“b”,”c”],[“bc”,”cd”]], values = [1.5,2.5,5.0], queries = [[“a”,”c”],[“c”,”b”],[“bc”,”cd”],[“cd”,”bc”]]
输出：[3.75000,0.40000,5.00000,0.20000]

示例 3：

输入：equations = [[“a”,”b”]], values = [0.5], queries = [[“a”,”b”],[“b”,”a”],[“a”,”c”],[“x”,”y”]]
输出：[0.50000,2.00000,-1.00000,-1.00000]

链接：https://leetcode-cn.com/problems/evaluate-division

算法及其分析

首先对于这个问题，我们可以将每个等式的两个字符串看作图的两个相邻顶点，且从起点到终点的边权重就是values中的值，终点到起点的边的权重就是这个值的倒数。如果一个给定的查询要有结果，那么查询的两个点之间一定存在一条路径。

DFS

我们递归地访问图中的顶点，并且记录下相应边的权重并计算，最后可以得到结果。注意，递归的编写，其 终止条件，即 递归边界 是十分重要的。对于图的数据结构，使用一个邻接表对于dfs来说是合适的：

加权并查集

当提到连通性时，并查集当然时一个好方法；从有向图的角度来说，本题的所有图的分量都是 强连通分量，这为并查集提供了有利条件。我们用一个额外的$dist$来记录节点到根的距离，这个距离定义为：
$$
d(x, root) ::=x/root
$$
其中$x/root$是本题中提到的除法运算；
