寻找安全的点

问题分析

在有向图中, 我们从某个节点和每个转向处开始, 沿着图的有向边走。 如果我们到达的节点是终点 (即它没有连出的有向边), 我们停止。

现在, 如果我们最后能走到终点，那么我们的起始节点是最终安全的。 更具体地说, 存在一个自然数 K,无论选择从哪里开始行走, 我们走了不到 K 步后必能停止在一个终点。

哪些节点最终是安全的？ 结果返回一个有序的数组。

该有向图有 N 个节点，标签为 0, 1, …, N-1, 其中 N 是 graph 的节点数. 图以以下的形式给出: graph[i] 是节点 j 的一个列表，满足 (i, j) 是图的一条有向边。

示例：
输入：graph = [[1,2],[2,3],[5],[0],[5],[],[]]
输出：[2,4,5,6]

提示：

graph 节点数不超过 10000.
图的边数不会超过 32000.
每个 graph[i] 被排序为不同的整数列表， 在区间 [0, graph.length - 1] 中选取。

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/find-eventual-safe-states

简单的分析就可以知道，对于图中的任何一个有向环中的顶点，它都不是安全的，因为其中的点作为起点，我们都可以在环中执行无限次的操作，所以本题是有向图判环的典型问题。而有向图的环的判定，常常联系到 拓扑排序。

知识分析

首先，通过dfs算法对图中点的访问，我们有如下定义：

  • 若访问的顶点在递归调用发生前加入队列，这被称为图中点的 前序排列
  • 若访问的顶点在递归调用发生后加入队列，这被称为图中点的 后序排列
  • 若访问的顶点在递归调用发生后加入栈，这被称为图中点的 逆后序排列

而拓扑排序其实就是点的逆后序排列，证明如下：

若图中存在路径v->w，那么当我们调用dfs(v)时，dfs(w)有如下情况：

  • dfs(w)已经调用并且返回，这个时候w被标记；
  • dfs(w)还未被调用，但是因为存在路径v->w，最终它会被直接或者间接的调用，并在dfs(v)之前返回；
  • dfs(w)已经调用并且还未返回，这表明有一条w->v的路径，所以图中存在环而无法实现拓扑排序

在我们假设图是有向无环图的时候，可以看出，后序排列中w在v前，而逆后序排列中，就是符合拓扑排序的定义的排列了：v在w前；对于逆后序排列，一个常见的办法是对图的反向图找出后序排列，这里我们也会遇到这个思想。

算法及其分析

基于上面的知识分析，可以写出下面两个算法：

拓扑排序

我们将所有的边反向，得到反向图 rgraph，随后将 rgraph 中所有没有入边的节点加入队列中。每一次我们取出队列中的一个节点 u，将它从图中删除，如果此时某个节点 v 存在从 u 到 v 的一条边，并且在删掉了这条边后，v 变成了没有入边的节点，那么就把 v 加入队列。以此类推，直到队列为空。最后所有加入过队列的节点即为安全的节点。

时间复杂度：$O(|V| + |E|)$
