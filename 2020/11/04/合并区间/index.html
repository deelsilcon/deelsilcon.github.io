分析

当数组中的区间是毫无顺序的时候，本题还是有一定难度的，但是如果区间的左端点是有序排列的，这个问题就会变得很简单；假设数组元素已经如上所称是有序的，那么对于每一个新的区间，都只需要用它的左右端点与现在的区间进行判断合并即可；

而且可以证明：

可以合并的区间都是连续的

证明：在排完序后的数组中，两个本应合并的区间没能被合并，那么说明存在这样的三元组 $(i, j, k)$以及数组中的三个区间 $a[i]$,$ a[j]$,$a[k]$ 满足 $i < j < k$ 并且 $a[i],a[k$] 可以合并，但 $a[i],a[j]$和$a[j],a[k]$ 不能合并。这说明它们满足下面的不等式：

$a[i].end < a[j].start \quad (a[i] \text{ 和 } a[j] \text{ 不能合并}) \ a[j].end < a[k].start \quad (a[j] \text{ 和 } a[k] \text{ 不能合并}) \ a[i].end \geq a[k].start \quad (a[i] \text{ 和 } a[k] \text{ 可以合并}) \
$

我们联立这些不等式（注意还有一个显然的不等式 $$a[j].start \leq a[j].end）$$，可以得到：

$$a[i].end < a[j].start \leq a[j].end < a[k].start$$，产生了矛盾！这说明假设是不成立的。因此，所有能够合并的区间都必然是连续的。

算法

有了上述分析，算法的设计就变得很简单了，我们进行按左端点大小对区间进行快速排序，再对得到的区间序列进行一次遍历，遍历中只可能有两种情形：

  • 遇到的区间与已知的区间不相交，直接加入答案；
  • 遇到的区间与已知的区间相交，则合成区间右端点取两者较大值即可

实现

时间复杂度:$O(N +N\log N)$

由上面的问题，很容易推导出这个问题的答案，在给定的区间集合 $\mathcal{X}$ 互不重叠的前提下，当我们需要插入一个新的区间 $S = [\textit{left}, \textit{right}]$ 时，我们只需要：

  • 找出所有与区间 S 重叠的区间集合 $\mathcal{X} $

  • 将 $\mathcal{X}$ 中的所有区间连带上区间 $S$ 合并成一个大区间；

  • 最终的答案即为不与 $\mathcal{X} $重叠的区间以及合并后的大区间。
