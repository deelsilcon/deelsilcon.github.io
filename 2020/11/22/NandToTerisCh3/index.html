Memory

概述

这一章中的问题中，出现了一种新的抽象：时序逻辑（sequential logic)，它不同于组合逻辑（输出仅仅是输入的一个函数映射值），其输出取决于历史的输入（即对内存的影响）以及当前的输入（可选是否载入内存）。对于这类逻辑门能保持其历史输入的这一功能，我们描述为它们能保持它们的状态。

重要内容

时钟单位

在大多数计算机中，时间的流逝由主时钟表示提供连续的交替信号序列。确切的硬件实现通常基于一个振荡器，该振荡器在两个振荡器之间连续交替标记为0–1，低高等的相位。从一次0-1变化开始到结束的时间间称为周期，每个时钟周期用来模拟一个离散时间单位。当前时钟相位（刻度）是用二进制信号表示。使用硬件电路，此信号将同时广播到整个计算机平台上的每个顺序芯片。这就是我们经常所说的几个时钟信号（several clocks)的来由，这是一种将连续的不可分的时间化成一种离散可描述的抽象和实现。

触发器

最简单的状态保持门是触发器（flip flop)，它有一个bit的输入和一个bit的输出；它的输入和输出的时间关系很简单：out(t)=in(t-1)，其中t为第t个clock。

图示的是D类触发器（data flip flop)，是较为简单的一种。我们通过触发器和之前用Nand建立的组合逻辑门，就可以实现更复杂的时序操作；

寄存器

寄存器是一种可以“存储”或“记住”值的存储设备，随着时间的流逝，它能实现基本的存储行为；
另一方面，DFF这样的触发器只能输出其先前的输入。但它们都含有对先前输入处理这样的时序概念，所以一个简单的寄存器可以由触发器得到：

随机存储单元

随存（Random Access Memory RAM）的名字来源于我们对任何地址（无论其物理实际位置）的访问都需要$O(1)$时间这一要求；我们希望对于每个地址的访问都是等时的。通过将上面的最简单的寄存器组合在一起，就能实现一个随机存储单元。

程序计数器

典型的CPU包括一个程序计数器，其输出被解释为当前程序中下一条应该执行的指令的地址。
一个计数器芯片可以通过组合一个输入/输出逻辑和一个具有组合逻辑的标准寄存器（用于添加常量）来实现。通常，计数器将必须配备一些其他功能，例如将计数重置为零，加载新的计数基数或递减而不是增加。

PC

PC的实现可以说是最具有挑战性的，需要注意以下几点：

  • 由上图中可以看到，三个操作是具有逻辑上的先后顺序的，即：
    • 如果reset被置位，则inc和load被忽略；
    • 如果inc被置位且reset未置位，load被忽略；
    • 仅当inc和reset都没有被置位时，load才被进行判断；
  • 不要仅仅以组合逻辑的思路来思考，而要用时序逻辑来思考
    • 怎么样建立这次输出和上次输出的关系？
    • 什么时候才载入in？
