Computer Architecture

1.基础

1.1 储存程序的理念

储存程序的理念（stored program concept)在计算机体系结构中尤为重要：计算基于固定的硬件平台，能够执行非常简单的固定的指令。 同时，这些指令可以组合在一起，产生任意更为复杂的程序。而且，这些程序的逻辑并未像1930年之前的机械计算机中那样嵌入硬件。程序的代码就像数据一样，暂时存储和操纵在计算机的内存中。存储程序的理念是许多抽象的实用计算机模型的关键要素，最著名的是通用图灵机（1936年）和冯·诺依曼机（1945年）。图灵机用来描述简单的计算机的抽象组件，主要用于在理论计算机科学中，用于分析计算的逻辑基础问题和解决方案。相比之下，冯诺依曼机是一种更实际的架构，可以说是当今几乎所有计算机平台的蓝图。下图是一个冯诺依曼机的图示：

1.2 内存

像其他硬件元素一样，可以从物理角度讨论存储单元 也可以从逻辑的角度来看。从物理上讲，内存是可寻址的线性序列的寄存器，每个都有唯一的地址和一个值，该值是固定大小的信息字（word)。从逻辑上讲，内存分为两个区域。一个区域专用于存储数据，例如:当前正在执行的程序的数组和对象;而另一个区域专门用于存储程序的指令。尽管所有这些“数据字”和“指令字” 在物理上看起来完全一样，但它们的目的却截然不同。在实际设计中，储存这两种指令的内存可以是一个物理上的实际位置，也可以是两个物理上分离的位置。这两种设计都有它自己的好处。

所有内存寄存器（无论其作用如何）均以相同的方式访问：为了操作特定的内存寄存器，必须首先通过提供一个地址。此操作可使我们立即访问寄存器的数据。随机访问存储器（RAM）通常用于强调这一重要的事实：每个随机选择的 可以在相同的访问时间内访问寄存器，而不管内存的大小和寄存器在其中的位置。

1.2.1 数据内存

高级程序处理抽象的组件，例如变量，数组和对象。将程序翻译成机器语言后，这些数据抽象变成二进制代码，存储在计算机内存中。一旦一个寄存器通过指定其地址从内存中选择，我们就可以读取或写入内容。

1.2.1 指令内存

在计算机上执行高级程序之前，它们必须被翻译成机器语言。翻译之后，每个高级语言的语句变成一系列一个或多个机器语言指令。这些指令存储在计算机的指令内存的形式为二进制代码。在程序执行的每个步骤中。CPU从指令中的选定寄存器中提取（即读取）二进制机器指令并对其进行解码，执行指定的指令，并找出下一个要执行的指令，执行下一步操作。

1.3 CPU

CPU是计算机体系结构的核心部分，负责执行当前加载程序的指令。这些指令告诉CPU需要计算什么，必须读取或写入哪个寄存器以及必须执行哪个指令，最后找出下一个指令的地址并执行下一步。 CPU使用以下三个主要硬件元素执行这些任务：算术逻辑单元（ALU），一组寄存器和一个控制单元。

1.3.1 ALU

算术逻辑单元：ALU用于执行计算机所具有的所有低级算术和逻辑运算。例如，典型的ALU可以将两个数字相加，对两个数字进行按位“与”运算，比较两个数字等等。一个ALU应该具有多少功能取决于需求、预算、能源和类似的成本效益方面的考虑。 LU不支持的任何功能（作为原始硬件操作）都可以在以后由计算机的系统软件来实现（当然，实现起来后速度相较之更慢）。

1.3.2 寄存器

寄存器：由于CPU是计算机的核心，因此它必须尽可能提高性能。为了做到这一点，我们将计算产生的中间结果储存在本地靠近ALU的地方，而不是频繁地将其输出和输入。因此，CPU通常是配备2组至32组小型的高速寄存器，每个寄存器可容纳一个一个字的大小。

1.3.3 控制单元

控制单元： 计算机指令以二进制代码表示，通常为16、32或64 位宽。在执行此类指令之前，必须先对其进行解码，然后将其中的信息导向各种需要控制位的硬件设备（ALU、寄存器、存储器等）。并且通过内部的程序计数器（program counter)，可以实现对程序的分支跳转等相关的操作。
